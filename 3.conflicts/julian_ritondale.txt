 
# 3. Conflicts

### 3.1. Preguntas

¿Qué es un conflicto? Cuando ocurre? ¿Es bueno o malo?


Un conflicto es una instancia o situacion en donde diferentes commits contienen cambios distintos de un mismo
archivo. Esto es un problema porque git debe combinar los branches y escoger una unica version. No es algo bueno
ni malo, sino natural. Sin embargo, es mas deseable recibir un conflicto antes de que git escoga una version por
su cuenta.


¿Se puede evitar un conflicto? ¿Cómo?


Los conflictos se evitan tomando iniciativas de organizacion de equipo.
