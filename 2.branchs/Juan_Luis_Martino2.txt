2.1.1 Un branch es una rama que tiene git. Pueden haber varias y luego se pueden unir.
2.1.2 Pueden ser muy utiles porque, en casos como estos, se pueden dividir en diferentes ramas y tener diferentes progresos en cada una de ellas. Incluso se pueden unir.
2.1.3 Una branch se crea con el comando git branch "nombre_de_la_nueva_rama_".
2.1.4 Para cambiar a otra rama se utiliza el comando "git checkout "nombre"".
2.1.5 Para eliminar una branch se utiliza el comando "git branch -d "nombre_de_la_rama".
2.1.6 Se puede utilizar el comando "git checkout -b "nombre_de_la_nueva_rama" para crear una nueva rama y directamente ir a ella.
2.1.7 Un merge es la union de 2 o mas ramas en una sola.
2.1.8 Para hacer un merge, se utiliza el comando "git merge".
2.1.9 Un tag es	un punto referencia en un punto especifico de la historia del repositorio.
2.1.10 Para crear un tag se utiliza el comando "git tag "x"".
