Teorica:

2.1.1) Un branch es una "rama" que se sépara de la rama principal, estas se usan para trabajar en distintas areas del mismo proyecto de forma aislada antes de fusionarlo con la rama principal (main).

2.1.2) Pueden ser utiles para, al momento de realizar un proyecto, en caso de querer realizar una prueba o querer modificarlo de forma aislada sin afectar al mismo, se puede crear esta y si estamos conformes con los cambios, fusionarlo al main.

2.1.3) Una branch se realiza con el comando Git Branch "Nombre de la branch" previamente verificando que estamos en la branch main.

2.1.4) Para cambiar a una branch se usa el comando git branch checkout "Nombre de la branch".

2.1.5) Para eliminar una branch se debe usar el comando Git Branch -d "Nombre de la branch"

2.1.6) Para crear una branch y cambiar a ella automaticamente se agrega -b al comando, quedando asi: Git Branch -b "Nombre"

2.1.7) Merge es un comando que junta una branch con otra fusionandola y combinando la diferencia entre ambas dando como resultado una branch que contiene todos los cambios de ambas.

2.1.8) Se realiza el comando git merge <Nombre de la branch que se quiere fusionar con la actual> verificando previamente que estamos en la branch que deseamos fusionar.

2.1.9) Un tag es una etiqueta con la que se marca una version especifica de un archivo que se va a necesitar posteriormente en este repositorio.

2.1.10) Se usa el comando git tag <nombre de la tag>
