1. Un branch es una copia independiente de un conjunto de archivos en un repositorio.
2. Los branches son útiles para trabajar en diferentes características, correcciones de errores o mejoras en paralelo y sin interferir con otros desarrolladores.
3. Para crear una branch en Git, se utiliza el comando git branch <nombre-de-la-rama>.
4. Para cambiar a una branch existente, se utiliza el comando git checkout <nombre-de-la-rama>.
5. Para eliminar una branch en Git, se utiliza el comando git branch -d <nombre-de-la-rama>.
6. Para crear y cambiar a una branch en un solo paso, se utiliza el comando git checkout -b <nombre-de-la-rama>.
7. Un merge es el proceso de combinar dos o más branches o versiones de un proyecto en una sola.
8. Para realizar un merge en Git, primero se cambia a la rama destino y luego se utiliza el comando git merge <rama-a-mergear>.
9. Un tag es una etiqueta que se le asigna a un punto específico en la historia del repositorio, como una versión o una release importante.
10. Para crear un tag en Git, se utiliza el comando git tag <nombre-del-tag>, y es posible agregar un mensaje con el comando `git tag -a <nombre-del-tag> -m "<mensaje-del-tag>".
