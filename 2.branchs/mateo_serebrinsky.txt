Una branch es una línea de desarrollo separada en Git que tiene sus propios commits y cuenta una historia diferente.
Las branches sirven para experimentar sin afectar el trabajo principal y para eliminar las que ya no se necesitan.
Git branch crea una nueva branch.
Git checkout cambia de branch.
Git branch -d borra una branch.
Git checkout -b crea y cambia a una nueva branch.
Git merge combina branches separadas en una sola línea de desarrollo.
Git merge junta las branches.
Git tag es una etiqueta que señala un punto específico en el historial del proyecto.
Git tag crea una etiqueta.
