
2.1 preguntas
1. ¿Qué es un branch?
Es un grupo de commits que crea una sola narrativa
2. ¿Por qué pueden ser útiles los branches?
Son utiles proque permiten crear listas o eliminar ramas del sistema segun sea necesario
3.¿Cómo se crea una branch?
Para crear una branch hay que usar el comando git branch
4.¿Cómo se cambia a una branch?
Usando el comando git checkout
5.¿Cómo se elimina una branch?
Usando el comando git branch -d
6.¿Cómo se crea una branch y se cambia a ella en un solo paso?
Usando el comando git checkout -b
7.¿Qué es un merge?
Un merge es es la forma que tiene Git de volver a unir un historial bifurcado. El comando git merge permite tomar las líneas independientes de desarrollo creadas por git branch e integrarlas en una sola rama.
8.¿Cómo se realiza un merge?
Se realiza un merge usando el coamndo git merge
9.¿Que es un tag?
Los tag son etiquetas de referencia que apuntan a puntos concretos del historial de git
10.¿Cómo se crea un tag?
Ejecutando el comando git tag
