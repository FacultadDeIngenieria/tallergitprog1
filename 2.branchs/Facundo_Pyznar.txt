2.1

1. Un branch es una copia independiente de un repositorio que permite trabajar en cambios aislados.
2. Los branches son útiles para trabajar en diferentes características o correcciones sin afectar el código principal.
3. Para crear un branch se utiliza el comando "git branch nombre_del_branch".
4. Para cambiar a un branch se utiliza el comando "git checkout nombre_del_branch".
5. Para eliminar un branch se utiliza el comando "git branch -d nombre_del_branch".
6. Para crear y cambiar a una nueva branch en un solo paso se utiliza el comando "git checkout -b nombre_del_branch".
7. Un merge es una operación que combina los cambios de un branch en otro.
8. Para realizar un merge se utiliza el comando "git merge nombre_del_branch".
9. Un tag es una referencia estática a un punto específico en el historial de versiones de un repositorio.
10. Para crear un tag se utiliza el comando "git tag nombre_del_tag".
