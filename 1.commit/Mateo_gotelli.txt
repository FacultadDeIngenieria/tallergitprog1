1.1. Preguntas
Para inicializar un repositorio local, se debe ejecutar el comando git init. Esto creará un nuevo repositorio en el directorio actual y configurará la estructura necesaria de Git para comenzar a rastrear los cambios en los archivos.

Para que un directorio deje de ser controlado por Git, se debe ejecutar el comando git rm --cached <directorio>. Esto eliminará el directorio del índice de Git, pero no eliminará los archivos físicamente del sistema de archivos.

Si se agrega un archivo a un directorio que ya está siendo controlado por Git, el archivo no será controlado automáticamente por Git. Es necesario utilizar el comando git add <archivo> para agregar explícitamente el archivo al área de preparación (staging area) y comenzar a rastrear los cambios en él.

El comando utilizado para agregar un archivo al repositorio local es git add <archivo>. Este comando agrega el archivo especificado al área de preparación (staging area), lo que significa que los cambios en ese archivo se incluirán en el próximo commit.

Para determinar qué archivos han sido modificados en el repositorio, se puede ejecutar el comando git status. Este comando mostrará una lista de los archivos modificados en comparación con la versión anterior y también indicará los archivos que están en el área de preparación (staging area).

El comando utilizado para hacer un commit es git commit -m "mensaje del commit". Este comando toma todos los cambios en el área de preparación y los guarda como un nuevo commit en el historial del repositorio, junto con un mensaje descriptivo que explica los cambios realizados.

En palabras simples, un commit es una acción en Git que guarda un conjunto de cambios realizados en los archivos del repositorio en un momento específico.
